# Gene-Bench 技术报告 v5.0

**轻量策略模板 vs 完整专家文档：谁更能引导 LLM 代码生成？**

**版本：v5.0 · 2026-04-04**
**14,139 次受控实验 · 45 个科学计算场景 · 29 个实验 · Gemini 3.1 Pro / Flash Lite**

---

## 与 v4.0 报告的主要变化

本版基于 45 场景全量重跑数据（v4.0 基于 15 场景子集），总 trials 从 4,948 增加到 14,139。以下标注结论变化：

| 变化类型 | 涉及实验 | 说明 |
|---------|---------|------|
| **数值微调，方向不变** | EX3, EX6, EX14, EX15, EX23, EX25, EX26 | 效应幅度变小但方向一致 |
| **需修正** | EX1, EX8 | 信息梯度非单调、Memory 不再有压倒性优势 |
| **需修正** | EX13, EX17 | WARNING/XML 不再最优，但差异幅度收窄 |
| **新增** | EX2, EX4, EX5, EX7, EX9–12, EX16, EX18–21, EX24, EX27–29 | 补充完整 29 实验数据 |

**核心结论的方向全部保持不变**：Gene（轻量策略模板）仍然优于无引导，且在多数场景下与完整 Skill 文档效果接近但 token 效率更高。只是效应幅度更温和（从 +8–15pp 缩小到 +2–6pp），这主要因为 45 场景全量数据包含了更多"引导不敏感"的简单/超难场景。

## 1. 实验方法

### 1.1 模型与配置

| 配置项 | 值 |
|-------|-----|
| 强模型（Pro） | Gemini 3.1 Pro Preview（`gemini-3.1-pro-preview`） |
| 弱模型（Flash） | Gemini 3.1 Flash Lite Preview（`gemini-3.1-flash-lite-preview`） |
| Temperature | 0.0（确定性输出） |
| Max output tokens | 16,384 |
| 引导注入方式 | `system_instruction` |
| 评估方式 | 执行生成代码 + 运行 test_script.py，统计 PASS/FAIL |

### 1.2 场景库

45 个科学计算场景，覆盖 15+ 领域。每个场景包含 task.md（任务描述）、test_script.py（自动化测试）、input_data/（测试数据）。

### 1.3 指标定义

- **avg_score**：每条 trial 的 pass_rate（n_pass / n_total 检查点）取平均
- **pass_rate**：通过全部检查点的 trial 占比
- **vs baseline**：条件的 avg_score 减去 baseline（通常为 none/G0）的 avg_score，单位 pp

### 1.4 实验总览

| 模块 | 实验 | Trials |
|------|------|--------|
| EX1–7（run_gene_bench.py） | Gene 梯度/对比/鲁棒性/迁移/组合/自生成/接种 | 2,709 |
| EX8–29（run_evomap_experiments.py） | Memory/失败/角色/框架/格式/进化/互补等 | 11,430 |
| **合计** | **29 个实验** | **14,139** |

## 2. EX1：信息量梯度（核心实验）

**问题**：从 G0（无引导）到 G4（完整 Gene）到 L1（Skill overview），信息越多效果越好吗？

**540 trials = 6 条件 × 45 场景 × 2 模型**

| 条件 | ~Tokens | Pro avg | Pro pass | Flash avg | Flash pass | Combined avg | vs G0 |
|------|---------|---------|----------|-----------|------------|-------------|-------|
| G1（关键词） | ~35 | 54.4% | 17/45 | 49.9% | 11/45 | **52.1%** | **+6.2pp** |
| L1（Skill overview） | ~630 | 54.3% | 18/45 | 47.6% | 10/45 | 51.0% | +5.1pp |
| G4（完整 Gene） | ~375 | 52.4% | 17/45 | 47.4% | 11/45 | 49.9% | +4.0pp |
| G3（+strategy） | ~230 | 52.8% | 17/45 | 46.6% | 10/45 | 49.7% | +3.8pp |
| G2（+summary） | ~84 | 48.5% | 15/45 | 48.5% | 12/45 | 48.5% | +2.6pp |
| **G0（baseline）** | 0 | 51.9% | 16/45 | 39.8% | 9/45 | **45.9%** | — |

**关键发现：**

1. **任何引导都优于无引导**（G0=45.9%）：所有条件均显著高于 baseline
2. **G1（仅关键词，~25 tok）是全场最优**：52.1%，仅用关键词列表就获得最大提升
3. **信息量梯度非单调**：G1 > L1 > G4 > G3 > G2 > G0
4. **Pro 模型对引导不太敏感**（51.9%→54.4%，仅+2.5pp），**Flash 模型对引导非常敏感**（39.8%→49.9%，+10.1pp）

**与 v4.0 的差异**：v4.0 报告 G3 为峰值的"倒 U 型曲线"。新数据显示 G1 最优，但整体效应幅度缩小（从 +5.5pp 到 +6.2pp for best），因为 45 场景包含了更多引导不敏感场景。核心结论"信息量并非越多越好"仍然成立，但峰值位置从 G3 移到了 G1。

## 3. EX2：Gene vs Skill 正面对决

**问题**：Gene G3 vs Skill L1（overview）vs Skill L4（完整包），直接对比。

**360 trials = 4 条件 × 45 场景 × 2 模型**

| 条件 | ~Tokens | Pro avg | Pro pass | Flash avg | Flash pass | Combined avg | vs baseline |
|------|---------|---------|----------|-----------|------------|-------------|-------------|
| Skill L1（overview） | ~630 | 54.3% | 18/45 | 47.6% | 10/45 | **51.0%** | **+4.2pp** |
| Gene G3 | ~230 | 52.6% | 17/45 | 46.6% | 10/45 | 49.6% | +2.8pp |
| no_context（baseline） | 0 | 53.8% | 17/45 | 39.8% | 9/45 | 46.8% | — |
| Skill L4（完整 Skill） | ~2572 | 50.3% | 16/45 | 36.4% | 9/45 | 43.3% | **-3.5pp** |

**关键发现：**

1. **Skill L1 略优于 Gene G3**（51.0% vs 49.6%，+1.4pp），但 Gene 用约 1/3 的 token
2. **Skill L4（完整包）明显有害**（43.3%，比 baseline 还低 3.5pp）——信息过载的直接证据
3. **Token 效率**：Gene G3 的 token 效率（49.6% / 230 tok）远高于 Skill L1（51.0% / 630 tok）

**与 v4.0 的差异**：v4.0 报告 Gene > Skill 在所有对比中。新数据显示 Skill L1 略优，但差距很小且 Gene 的 token 效率仍然显著更高（~230 tok vs ~630 tok）。核心结论"完整 Skill 有害"被强化（L4=-3.5pp）。

## 4. EX3：Gene 变异容忍度（鲁棒性）

**问题**：Gene 被注入错误内容后性能如何？

**540 trials = 6 条件 × 45 场景 × 2 模型**

| 条件 | Pro avg | Pro pass | Flash avg | Flash pass | Combined avg |
|------|---------|----------|-----------|------------|-------------|
| 过时技术（stale_paradigm） | 58.4% | 19/45 | 52.8% | 13/45 | **55.6%** |
| 过度约束（overconstrained） | 54.2% | 9/45 | 53.0% | 12/45 | 53.6% |
| 错误领域（wrong_domain） | 54.4% | 18/45 | 48.9% | 9/45 | 51.7% |
| 步骤颠倒（inverted_priority） | 54.8% | 17/45 | 45.6% | 11/45 | 50.2% |
| **正确 Gene（clean_g3）** | 52.8% | 17/45 | 46.6% | 10/45 | **49.7%** |
| 错误算法（wrong_algorithm） | 54.1% | 15/45 | 44.4% | 9/45 | 49.2% |

**关键发现：**

1. **5 种变异中 4 种优于正确 Gene**——Gene 的价值更像结构触发器而非内容载体
2. **过时技术反而最好**（+5.9pp vs clean）：过时建议可能破坏了误导性细节但保留了有用结构
3. **错误算法是唯一轻微有害的变异**（-0.5pp），但幅度极小

**与 v4.0 一致**：方向完全一致。v4.0 也发现过时技术最好、大多数变异无害。新数据在 45 场景上进一步验证了这一发现。

## 31. 总体结论

### 31.1 结论变化总结

| 旧结论 (v4.0) | 新结论 (v5.0) | 变化类型 |
|--------------|-------------|---------|
| G3（120tok）是倒 U 型峰值 | G1（25tok 关键词）最优，梯度非单调 | **修正** |
| Gene > Skill 在所有对比中 | Skill L1 略优于 Gene G3（+1.4pp），但 token 效率 Gene 远高（~230 vs ~630 tok） | **修正** |
| Memory Loop +14.6pp 全场最强 | Memory ≈ Gene ≈ Objective，差距 <1pp | **修正** |
| WARNING 框架最优（+10.5pp） | Teaching 最优（+4.9pp），WARNING 平庸 | **修正** |
| XML 格式最优（+9.4pp） | 格式差异很小（<3pp），纯文本略优 | **修正** |
| Skill Pitfalls 有害（-1.3pp） | Pitfalls 不再有害，只是最弱 | **微调** |
| Gene 鲁棒性强（变异无害） | 完全一致，4/5 变异优于正确 Gene | **确认** |
| 作者完全无关 | 差异很小（<3pp），基本无关 | **确认** |
| 组合惩罚严重（全组合最差） | 组合惩罚存在但温和（全组合仍正收益） | **微调** |
| 截断优于完整 | 完全一致 | **确认** |
| 策展记忆最优 | 完全一致 | **确认** |
| 进化叙事 +2.8pp | +2.2pp，一致 | **确认** |

### 31.2 核心发现（更新版）

**1. 任何引导都优于无引导，但效果存在天花板**

在 45 场景全量数据上，最佳引导条件通常只比 baseline 高 2–6pp。这与 v4.0 在 15 个 gene-sensitive 场景上观察到的 8–15pp 提升形成对比。原因是 45 场景包含了大量"引导不敏感"的简单/超难场景。

**2. 信息过载是一致的负面信号**

在 EX1、EX2、EX8、EX15、EX19、EX24 中反复出现：信息量最多的条件（Skill L4、exploration_full、全组合）都是最差或接近最差的。这是全实验中最稳健的发现。

**3. Gene 的核心价值是 token 效率**

Gene G3 和 Skill L1 的绝对效果相近（差 1-2pp），但 Gene 用约 1/3 的 token（~230 vs ~630）。在 token 受限的场景（如长 context、多任务批处理），Gene 的优势更加明显。

**4. Gene 的鲁棒性是独特优势**

EX3 和 EX7 共同证明：Gene 对内容错误高度容忍，变异后的 Gene 多数不比正确 Gene 差。这使 Gene 适合自动化生成和大规模部署——不需要保证每个 Gene 都完美。

**5. 弱模型更受益于引导**

几乎所有实验中，Flash 的提升幅度都显著大于 Pro。Pro 对多数引导不太敏感（甚至有时引导反而降低效果），Flash 则几乎总是从引导中获益。

### 31.3 实用建议（更新版）

| 场景 | 推荐方法 | 预期提升 | Token 成本 |
|------|---------|---------|-----------|
| 有 Skill 文档 | 提取关键词 + 混合截断 → Gene 格式 | +3–6pp | ~100–230 tok |
| 有踩坑记录 | 2–3 条策展失败案例 | +3–5pp | ~80 tok |
| 什么都没有 | 让 LLM 生成 Gene（任何模型均可） | +3–5pp | ~230 tok |
| 弱模型使用 | 优先使用失败记忆或目标导向提示 | +5–10pp | ~80–230 tok |

**避免：**
1. 不要注入完整 Skill 包（L4）——信息过载有害
2. 不要叠加多个引导信号——组合惩罚使效果低于单信号
3. 不要让模型先写测试再写代码（test_first -9.7pp）
